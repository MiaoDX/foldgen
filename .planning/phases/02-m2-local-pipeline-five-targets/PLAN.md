# Phase 2 Plan: M2 Local Pipeline

## Goal

Run five curated public target fixtures through a deterministic local pipeline
that produces inspectable output artifacts and records proposal/critic history
for every case.

## Tasks

1. Expand curated target fixtures from three to five with metadata.
2. Add a `foldgen-agent` package for deterministic Stage 1 pipeline logic.
3. Implement target profile selection from curated targets to public base forms.
4. Generate candidate local fold operations for each target without live model
   calls.
5. Validate every candidate and record failures in proposal/critic history.
6. Select the best valid candidate per case and write derived FOLD, crease SVG,
   validation JSON, diagram-step JSON, proposal history, critic history, and
   case summary artifacts.
7. Add a repo-local `npm run m2:pipeline` command that writes to
   `out/m2-pipeline/`.
8. Add tests proving the five-case batch output shape, validation status,
   history files, and failure recording.
9. Update README/status docs with the M2 workflow and verification.

## Constraints

- No live provider adapter, paid API, private repo, or private asset runtime
  dependency.
- Keep `fold-core` deterministic and free of agent policy.
- Treat critic scores as deterministic inspection metadata, not human
  reproducibility evidence.
- Do not build the M3 web demo in this phase.

## Expected Outputs

- `packages/foldgen-agent` implementation and tests.
- Five target fixtures under `benchmarks/targets/`.
- `out/m2-pipeline/` generated by the local command and ignored by git.
- README/status/planning updates for the M2 command.
